RENDER DEPLOYMENT INSTRUCTIONS FOR BACKEND
==========================================

IMPORTANT: If you get "gunicorn: command not found" error, see RENDER_TROUBLESHOOTING.txt

1. Create a Render account at https://render.com

2. Click "New +" and select "Web Service"

3. Connect your Git repository or upload the backend folder directly

4. Configure the service:
   - Name: merchant-copilot-api
   - Environment: Python 3
   - Root Directory: backend (or merchant-intelligence-copilot-files/backend if monorepo)
   - Build Command: pip install -r requirements.txt
   - Start Command: python -m gunicorn app:app --bind 0.0.0.0:$PORT
   - Instance Type: Free or Starter

5. Add Environment Variables (in Render dashboard):
   Required:
   - AWS_ACCESS_KEY_ID: your_aws_access_key
   - AWS_SECRET_ACCESS_KEY: your_aws_secret_key
   - AWS_REGION: ap-south-1
   
   Optional (defaults provided):
   - BEDROCK_MODEL_PRIMARY: amazon.nova-pro-v1:0
   - BEDROCK_MODEL_FAST: amazon.nova-lite-v1:0
   - BEDROCK_MODEL_BASELINE: amazon.nova-micro-v1:0
   - TEMPERATURE: 0.2
   - MAX_TOKENS: 500
   - TOP_P: 0.9
   - APP_ENV: production
   - LOG_LEVEL: INFO

6. Click "Create Web Service"

7. Wait for deployment to complete (5-10 minutes)

8. Copy your Render URL (e.g., https://merchant-copilot-api.onrender.com)

9. Update frontend .env with this URL

TROUBLESHOOTING:
If deployment fails with "gunicorn: command not found":
- See RENDER_TROUBLESHOOTING.txt for detailed solutions
- Quick fix: Change Start Command to: python app.py

DIRECT UPLOAD METHOD:
- Zip the backend folder
- Upload to Render via "Manual Deploy" option
- Follow steps 4-8 above

Files included for deployment:
- app.py (Flask application)
- requirements.txt (Python dependencies)
- Procfile (Render start command - uses python -m gunicorn)
- runtime.txt (Python version)
- render.yaml (Render configuration)
- build.sh (Build script)
- src/ (application code)
