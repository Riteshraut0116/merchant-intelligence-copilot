DEPLOYMENT METHODS:
-------------------

METHOD 1: Direct Upload (Easiest)
Backend:
1. Zip the entire backend/ folder
2. Upload to Render via dashboard
3. Configure environment variables
4. Deploy

Frontend:
1. Run: npm install && npm run build
2. Drag dist/ folder to Netlify drop zone
3. Done!

METHOD 2: Git Integration (Recommended for teams)
Backend:
1. Push backend/ folder to Git repository
2. Connect repository in Render
3. Configure build settings
4. Auto-deploy on push

Frontend:
1. Push frontend/ folder to Git repository
2. Connect repository in Netlify
3. Configure build settings
4. Auto-deploy on push

METHOD 3: CLI Deployment
Backend:
1. Install Render CLI
2. Run: render deploy

Frontend:
1. Install Netlify CLI: npm install -g netlify-cli
2. Run: netlify deploy --prod

ENVIRONMENT VARIABLES NEEDED:
------------------------------
Backend (.env or Render dashboard):
- AWS_ACCESS_KEY_ID (required)
- AWS_SECRET_ACCESS_KEY (required)
- AWS_REGION (default: ap-south-1)

Frontend (.env before build):
- VITE_API_BASE_URL (required, your Render URL)

DEPLOYMENT ORDER:

5. Test complete application

TROUBLESHOOTING:
----------------
Missing files error:
- Check this list and verify all files present
- Ensure folder structure matches

Build failures:
- Check requirements.txt/package.json
- Verify Python/Node versions
- Review build logs

Runtime errors:
- Check environment variables
- Review application logs
- Verify AWS credentials
